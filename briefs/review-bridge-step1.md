# Ревью — мост Шаг 1. Вердикт: НЕ мержим (живой тест поймал краш + ветка не изолирована)

**От:** workspace-архитектор + user. PR #12.

## Что хорошо
- Код моста сильный: **self-эхо отрезан** (author_id != self), resilience-loop с реконнектом/backoff,
  ошибки хода не роняют процесс, `claude -p` рантайм, аккуратный chater-клиент (Bearer **заголовком** на ws —
  верно, мост не браузер). Логика отделена от сокетов (тестируемо). Юнит-тесты + CI (ruff/pytest/CodeQL) зелёные.
- Мост **самодостаточен** — не импортит kernel/backend (проверил).
- 3 твоих вопроса — все ОК: `packages/bridge` как временный дом ✓; `claude -p` для Шага 1 ✓; формат промпта ✓.

## БЛОКЕРЫ (до мержа)
1. **Краш на 409 — поймано живым тестом.** `ensure_user` делает `POST /chater/users`; если бот уже существует →
   **409 Conflict** → `raise_for_status()` → мост падает на старте. Юзеры в chater persist'ятся → **мост не
   переживает рестарт** (второй запуск всегда 409). Юнит-тесты застаблены на chater-клиенте, поэтому пропустили —
   ровно то, что ловит живой прогон.
   - Фикс: `ensure_user` идемпотентен (409 = «юзер есть» = ок, не краш).
   - Нюанс: тебе нужен свой `author_id` для self-фильтра, а на 409 его в ответе нет. Два пути:
     - **(рекомендую) фильтровать self-эхо по СВОИМ запощенным message-id**: `post_message` возвращает `id` —
       запоминай их и игнорь входящие кадры с этим id. Тогда `author_id` не нужен вовсе, 409 безвреден,
       **chater менять не надо**. (ws пушит только новые сообщения, так что рестарт-«забывание» id не проблема.)
     - либо кросс-продукт-заказ chater'у: `GET /chater/me` (вернуть authed-юзера) — но это отдельный бриф chater.
2. **Ветка не изолирована.** PR #12 несёт **11 коммитов**: 1 — мост, **10 — прежняя brainer-машина** (kernel-
   события, backend OpenAPI/approval-эндпойнты, manifest, gateway-parity), которых нет в `origin/main`. Ты
   ветвился от локального main, обогнавшего origin на 10 непушнутых коммитов — они и «приехали». Мост
   самодостаточен → **ребейзни один bridge-коммит на `origin/main`**, чтобы PR = только мост.
   Судьба тех 10 коммитов — **отдельное решение user'а** (это замороженная машина, не часть этой задачи).
3. **Semantic PR title — красный** (reusable `omnifield/devopser/.../pr-title.yml`). Прочитай лог чека
   (`gh run view --job=<id> --log`), приведи заголовок/scope в соответствие — вероятно «bridge» не в allowed-scopes.

## Ещё НЕ подтверждено
Реальный ответ агента **live не проверен** — краш случился ДО loop'а, а `claude -p` headless-смоук завис
(долгий старт). После фикса (1) обязателен живой прогон: бот в комнате → user пишет → **бот отвечает live,
одним сообщением, без петли**. Это финальный DoD Шага 1.
