## Решение: вариант A

Запрет «base.ts не трогать» снимаю для ЭТОЙ строки — он означал «не
перестраивать env-шов», а не «не менять значение дефолта». Дефолт — это
контент шва, и его законное место именно там:

```ts
const BASE: string = (import.meta.env?.VITE_API_BASE as string | undefined) ?? "/api/brainer";
```

- Один source of truth, одинаковая семантика в dev/build/test — ровно то,
  ради чего parity-флоу затевался (дефолт совпадает с prod-значением за
  gateway, env-override остаётся для нестандартных раскладок).
- B (vite define) отклонён: прячет конфиг в сборку, семантика test/build
  расползается. C отклонён: правка корневого gitignore-канона ради значения
  одной константы — несоразмерно.
- Коммент в base.ts обнови под новую реальность (dev-прокси больше нет,
  base = same-origin путь через gateway).
- Если какой-то тест утверждает URL без префикса — чини ожидание теста
  осознанно, НЕ возвращая дефолт "".

## Разблокировка e2e

Gateway УЖЕ поднят devopser'ом (`stacks/gateway`, :8080; маршруты 502 до твоих
фиксов — ожидаемо). Так что STOP по live-проверке снят: после кода+тестов
проверяй HMR/стрим через `http://localhost:8080/brainer/` сразу.
