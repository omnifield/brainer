## Признаю (твой фидбек верен)
Плагины (Cline/Roo/Continue/Kilo) **не читают наш OpenAPI** и говорят на своём протоколе — значит
«лицо цепляется по нашему OpenAPI» и «лицо = плагин» взаимоисключающи. И глубже: у агент-плагинов
**свой мозг**, а brainer — сам агент-система → поверх друг друга дерутся. v1 это упустил. Снято.

## Три строки intent (что ты просил)
1. **Цель теста через плагины = ХАРВЕСТ, не подключение.** Плагины — это «**выставка**» (аналогия user):
   смотрим, как индустрия структурирует session/stream/tool-events/approval, и **точечно фиксируем
   бест-практис форм**. Они **референсы, НЕ клиенты** brainer. Ни «транспорт», ни «движок целиком» —
   мы снимаем **каноничные формы**.
2. **Второе лицо = НАШ тонкий клиент** (VS Code-плагин / десктоп / дашборд), не плагины. Мозг в
   движке, лицо тонкое. OpenAPI-публикация — для **наших лиц + будущих интеграций**, не для чужих
   плагинов.
3. **Приоритет:** расцепка back↔front = быстрая разблокировка (пульт пока не делаем → почти
   бесплатно, снять формулировку «shapes shared with frontend»). **Настоящий заказ** = поднять API
   из BFF в **самостоятельный версионируемый контракт** + формы **каноничные под флагманов**
   (харвест) ради будущих интеграций. Своё лицо — потом.

## Модель (уточнена, blueprint v2 §13.6)
brainer = **агент-система** (как Claude Code): папка → воркспейс; воркспейсов много; внутри —
**иерархия агентов** (workspace-архитектор → овнеры пакетов → помощники). Движок
(`kernel`/`channel`/`adapters`, провайдер-агностик) — **хорош, НЕ трогаем**.

## Что построить
- **Движок** — как есть.
- **API: BFF → самостоятельный контракт.** FastAPI OpenAPI = канон контракта (версия), расцеплен с
  дашбордом (дашборд = один потребитель).
- **Формы API/данных — каноничные** (принцип «стандарты на узлы, уникальная сборка», §13.5):
  харвест из Cline/Roo/Continue — как они лепят session/stream/tool-events → берём проверенные формы,
  собираем своё. Профит — будущие интеграции «поставил и работает».
- **Своё тонкое лицо** — отдельный трек позже (сейчас упор на API).
- **Бонус-опция (твой ADR, не обязалово):** Anthropic/OpenAI-совместимый стрим-эндпойнт → тонкий
  чат-клиент = дешёвое лицо. Семантику (passthrough «сырая модель» vs agent-over-completion) решаешь
  ты, ЕСЛИ возьмём. Сейчас не в критическом пути.
