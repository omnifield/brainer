| | |
|---|---|
| **Адресат** | brainer-архитектор → техпроработка → owner-брифы |
| **От** | оракул-архитектор, 2026-07-10 (решение user, обсуждено с ним) |
| **Тип** | переприоритизация, НЕ разворот: контракт kernel уже provider-native (answers-бриф §0, З1–З3) — теперь строим ПЕРВЫЙ native-провайдер |
| **Мотив user** | (1) риск глубоких подвязок на Claude API при активном тесте; (2) деньги — активные тесты на платном API накладны; (3) базовые мехи тестятся на локалках бесплатно; (4) агентов до стаба опасно пускать внутрь продукта — тестим рядом |

## Решения (user-confirmed)

### 1. Порядок перевёрнут
Self-hosted (локальный) провайдер kernel-контракта строится **ПЕРВЫМ** и становится
основной средой тестов мехов. **claude-code адаптер — FREEZE**: живой, работает,
reference-реализация контракта; не углублять и не расширять до стаба базы. Интеграции =
ориентир правильного API, не фундамент.

### 2. Conformance-suite — техническое оформление «ориентира»
Единый тест-набор ПОВЕРХ kernel-контракта (события: message/thinking/tool-call/tool-result/
status/done/error/limit/permission-request; операции: launch/send/stream/resume/stop),
гоняется против ЛЮБОГО провайдера одинаково:
- **fake** (детерминированный, CI, скрипт-сценарии) — быстрый и бесплатный;
- **local** (реальный инференс на локалке) — правда жизни бесплатно;
- **claude-code** (замороженный) — сверка «ориентира»: расхождение = либо баг адаптера,
  либо дыра контракта → к тебе.
Suite — часть kernel (контракт владеет своими тестами). Это ГЛАВНЫЙ деливерабл против
coupling-риска: мехи никогда не тестятся «через Claude», только через контракт.

### 3. Размещение: рядом, не внутри (прецедент chater/tasker)
Инкубация в brainer `packages/` как **extract-candidate** с жёстким фенсом (моя
рекомендация; user может решить отдельным репо — тогда правка одной строки):
- ноль импортов из продукт-пакетов (orchestrator/backend/frontend);
- тесты/прогоны — standalone (CLI-раннер против движка), продуктовый оркестратор и UI
  **не трогаются до стаба**;
- судьба — capability-ярус экосистемы (ADR 078 §3 оракула: движки = shared backend),
  extract по триггеру второго потребителя.

### 4. Состав работ (двухслойно, зеркало voice/image-паттерна)

**Слой A — инференс-движок (capability, тупой).** Порт из оракула `backend/llm`
(ADR 074): py3.12/FastAPI/uv, реестр движков (`LlmEngine` Protocol, lazy),
`POST /llm/generate {prompt, system?, schema?, max_tokens?, temperature?, engine?}` →
текст ЛИБО валидный JSON по схеме (llama.cpp grammar-enforcement — ключ к tool-call'ам
на малых моделях), движки `llama-cpp` (GGUF, путь весов env, air-gapped; GPU offload
3070 Ti) + `fake` (CI). **Порт делаю я (оракул)** — fix-then-transfer точечно, код
капсулы мой домен; тебе приношу готовый пакет под твой фенс. Паттерн сервиса тебе
знаком по voice/image капсулы — тот же.

**Слой B — agent-loop (мозг провайдера) — твоя техпроработка.** Реализует kernel-контракт
НАПРЯМУЮ (это native-провайдер, без адаптера): цикл prompt → generate (schema=tool-call |
text) → эмит событий контракта → исполнение tool → повтор; персона/системный промпт —
из роли пресета; permission-mode — режимом (permission-request событие эмитить умеет —
контракт уже зарезервировал); resume — свой state (реестр сессий уже персистентный).
Тулсет v0 — минимальный и типизированный (read/write/exec в песочнице прогона), словарь
рук расширяем позже (MCP-consume — отдельное решение, НЕ v0).

### 5. Правда о локалках (чтобы не разочароваться не туда)
1–4B модели ГЛУПЫЕ: они валидно дёргают руки (grammar спасает структуру), но решения
слабые. Это **фича теста мехов** — конформность, тайминги, устойчивость петли, не
интеллект. «Агент реально полезен» проверяется позже на claude-code/жирных моделях —
через тот же conformance-suite и пульт.

## Порядок

A (порт движка, я) ‖ conformance-suite против fake (ты, сразу — fake не ждёт движка) →
B agent-loop против fake → B против local → сверка claude-code через suite → отчёт user:
«мехи стабильны, что открываем в продукт».

## Вне скоупа
Углубление claude-адаптера · MCP-consume/produce · peer-провайдеры · роутер-тарифы ·
пускание агентов в продуктовые зоны brainer (до стаба и решения user).

## Координация
Blueprint слоя B (цикл, тулсет v0, схема tool-call) — показ user+мне ДО owner-брифов.
Расхождение контракта, вскрытое suite'ом, — жирный флаг сразу (это самое ценное, что
тест может найти).
