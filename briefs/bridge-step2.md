# brainer — мост Шаг 2: авто-джойн всех комнат агента (обнаружение)

Шаг 1 замержен (`main` 5381f34): один агент, **одна фиксированная** комната. Шаг 2 снимает фиксацию —
агент участвует **во всех комнатах, где он участник**, и подхватывает **новые**, куда его добавили, без `ROOM_ID`.
Делаешь ТОЛЬКО Шаг 2, ветка от свежего `main` (изолированно — не тащи припаркованные коммиты!), PR, STOP + отчёт.

## Проблема, которую решаем
Сейчас `ROOM_ID` обязателен и один. User добавляет бота в комнату — бот молчит, если это не та самая комната.
Нужно: **добавил агента в любой чат → он там отвечает.** (chater список комнат по ws НЕ пушит — поэтому
обнаружение = периодический поллинг `GET /chater/rooms`.)

## Что строим
- **Обнаружение комнат:** периодически (`ROOMS_POLL_S`, дефолт ~10с) звать `GET /chater/rooms` под
  `AGENT_HANDLE` → это комнаты, где агент участник (chater `ListRoomsForUser`). chater менять НЕ надо.
- **Supervisor подписок:** на каждую комнату — своя ws-подписка + Step-1 `Bridge` (переиспользуй как есть:
  self-эхо по своим message-id, один ответ, resilience). Новая комната в списке → поднять задачу/подписку;
  пропала → аккуратно закрыть. Конкурентно (asyncio-задача на комнату).
- **Конфиг:** `ROOM_ID` становится **необязательным** (если задан — режим одной комнаты, обратная
  совместимость Шага 1; если пуст — авто-режим по всем комнатам). Добавь `ROOMS_POLL_S`.
- **Self-эхо / без петли:** per-room, как в Шаге 1 (posted-id set на комнату). Никаких author_id.
- **Resilience:** падение поллинга или одной ws-подписки не роняет процесс и другие комнаты; новая комната
  подхватывается в пределах интервала поллинга; реконнект per-room.

## DoD Шага 2
- Старт **без `ROOM_ID`** → мост обнаруживает все комнаты агента и отвечает в каждой.
- **Живьём:** пока мост запущен, user добавляет `AGENT_HANDLE` в **новую** комнату и пишет → в пределах
  ~`ROOMS_POLL_S` бот присоединяется и **отвечает live**. Работает в **двух+ комнатах одновременно**, без петель,
  без перекрёстных ответов (ответ уходит в ту же комнату).
- Режим одной комнаты (`ROOM_ID` задан) продолжает работать (обратная совместимость).
- Тесты (застабленный chater-клиент): обнаружение новой комнаты → подписка; удаление → отписка; изоляция
  posted-id по комнатам. CI по канону brainer зелёный. Ветка + PR. STOP → отчёт.
