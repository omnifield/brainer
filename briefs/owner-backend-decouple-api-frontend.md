## Зачем (одной строкой)
API сейчас **co-designed с дашбордом** («shapes shared with the frontend; change only via
architect»). Пульт мы пока не строим → снимаем эту зависимость: **контракт становится первичным,
дашборд — просто один потребитель**. Формы не меняем — меняем их *статус и владение*.

## Границы (жёстко)
- **Движок не трогаем** — `kernel`/`channel`/`adapters` как есть.
- **Формы событий/эндпойнтов НЕ меняем** в этом брифе. Каноничные формы (харвест из плагинов) —
  **отдельный, следующий трек**; сюда не лезь, дождись контракт-брифа. Здесь только расцепка владения.
- Это **не** версионирование OpenAPI (тоже следующий трек). Только снять «форма под фронт».
- Упрёшься в cross-zone / изменение контракта → **STOP + эскалация к architect** (канон).

## Что сделать

### 1. Снять «shapes shared with frontend» как проектную зависимость
- `packages/backend/app/api/sessions.py` — докстринг: убрать формулировку *«The launch/stop/events/
  messages shapes are shared with the frontend; change only via architect»*. Заменить на честную
  модель владения: **контракт первичен и провайдер-/лицо-агностичен; дашборд — ОДИН потребитель
  опубликованного контракта, не co-owner формы**. Смысл: править контракт можно только через architect
  (это остаётся), но **не потому что «фронт завязан»**, а потому что это публичный контракт.
- Пройди `app/api/*` и `app/main.py` — вычисти любые другие формулировки/комментарии, где форма
  оправдана нуждами дашборда. Оправдание формы = «это контракт», не «это надо фронту».

### 2. Зафиксировать `kernel events.py` как единственный источник формы (уже так — закрепить)
- `packages/kernel/.../events.py` уже single-source (9 типов, snake_case, schema-ген). Убедись, что
  backend **нигде не дублирует** формы событий локально под фронт (легаси camelCase interface-mvp
  модели — если ещё живы и участвуют в текущем контракте, зафиксируй их наличие в отчёте, НЕ трогай,
  это отдельное решение architect).
- Проверь: генерация JSON Schema из `events.py` и drift-тест — зелёные и не зависят от фронта.

### 3. Отчёт о состоянии границы
Короткий `packages/backend/README.md`-раздел (или отчёт в PR): где сейчас backend знает про дашборд
(CORS/origin, пути, camelCase-легаси, что угодно), и что из этого — **обоснованная связь потребителя**
(ок), а что — **скрытая co-design зависимость** (кандидат на снятие). Это питает следующий трек.

## Чего НЕ делать
- Не добавлять/не убирать типы событий и поля. Не менять пути. Не вводить `/v1/`. Не публиковать
  OpenAPI-артефакт. Не удалять фронт-код. Всё это — следующие треки/другие зоны.

## DoD

  «контракт первичен, дашборд — потребитель».
- `events.py` подтверждён как single-source; backend не дублирует формы под фронт; drift-тест зелёный.
- Отчёт о границе (где backend ↔ дашборд) — сдан, разделено «обоснованная связь» vs «скрытая co-design».
- pre-commit (test+lint+build) зелёный. Движок и формы — байт-в-байт как были.
