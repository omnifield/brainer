## Что

1. **`_pump_sdk` не должен молчать на не-SDK исключениях.** Сейчас ловится только
   `ClaudeSDKError`; любое другое исключение закрывает стрим через `finally → closed`
   без `error`-события (раньше hub ловил это как `stream_crash` — регрессия
   видимости отказа). Фикс: `except Exception as exc → q.put(("error", exc))`
   (CancelledError — пробрасывать как сейчас).
2. **Упавший `query()` после инжекта echo.** `send()` кладёт `message {role:user}`
   в лейн до `client.query()`; если query бросил — в ленте остаётся «отправленная»
   реплика без хода. Порядок инжекта НЕ менять (гарантия «echo до ответа» важнее);
   фикс: исключение query → эмиттить в лейн `error` (retryable по типу ошибки)
   и пробрасывать наружу как сейчас (API уже отдаёт ошибку фронту).

3. Попутно (одной строкой): `hub._maybe_persist_sid` — заменить `getattr`-fallback
   прямым вызовом `self._adapter.current_handle(...)` ПОСЛЕ того, как owner-kernel
   закоммитит `fix-kernel-current-handle.md` (порядок: kernel первым; если kernel
   ещё не готов — пункт 3 пропустить, не блокируясь).
