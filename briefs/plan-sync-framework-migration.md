## План (решения user, 2026-07-11)

1. **Стартовала миграция web-фреймворка** (репо `weber`, веду я — оракул).
   Итерации: **0** скелет-финализация (пересадка на devopser-эталон + containers-only,
   клон в WSL2 FS) → **1** кор (`web-state` → `web-core` с аудит-фиксами CC-4/CC-10)
   → **2+** пакеты needs-driven (style → ui → router → query → …). Перед каждой
   итерацией — отдельное обсуждение ходов с user. Перенос точечный
   (fix-then-transfer, эталон weber ит.2), аудит оракула = таможня болячек, не роадмап.
2. **Зачем фреймворк первым — тонкие интерфейсы.** Вижн user: brainer
   декомпозируется на продукты — чат→**chater**, таск-менеджер→**tasker**,
   брифы/доки→**writer**, мониторинг→**grafer** (новый, имя рабочее). Интерфейсы
   всех продуктов = композиции из фреймворка и модулей, своей визуальной плоти
   минимум. Интерфейс brainer в перспективе пересобирается так же.
3. **Локалки ПАРКОВАНЫ** (не горят): порт `llm-engine` (мой) отложен, приоритет-флип
   local-agents-first не отменён, но сдвинут; claude-code остаётся рабочим
   провайдером пульта. Conformance-suite/слой B — без изменений, своим темпом,
   НЕ в ущерб роли пульта.
4. **Brainer-репо в тихом режиме**: user рулит агентами ЧЕРЕЗ brainer — конфликтующие
   работы в репо не стартуем, координация любых правок через user.

## Роль brainer сейчас = ПУЛЬТ (живой догфуд control-канала)

User управляет **weber-агентами** (миграция) и **chater-агентами** (см. ниже) через
brainer. Каждый день миграции = e2e-тест твоего продукта. Что от тебя:

1. **Консультировать user** по вопросам пульта: запуск/стоп сессий, headless-спавн,
   роли/скоупы (`OMNIFIELD_SCOPE`), permission-события, SSE/реконнект, credentials
   (секрет-volume `omnifield-secrets`), контейнер-раскладка (devbox, gateway :8080).
2. **Находки пульта = продукт-беклог brainer**: неудобство запуска, дыра в статусах,
   недостающее событие контракта — фиксируй у себя как фичи/баги продукта.
   Чинить weber/chater у себя — НЕЛЬЗЯ (их находки — брифом их архитекторам через user).
3. **Не удивляться chater-трафику**: chater-бэк (Go) готовится ПАРАЛЛЕЛЬНО
   агент-воркфлоу — founding-бриф кладу я в `chater/briefs/`, агентов запускает user
   через тебя. Интерфейс chater — позже, на фреймворке. Мост participant↔session —
   по своему брифу, не сейчас.
