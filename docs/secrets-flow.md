# Секреты — флоу для публичных репо

Все репо omnifield публичные (решение 2026-07-09). История git публична **навсегда** —
«удалить файл следующим коммитом» не работает. Отсюда правила.

## Принципы

1. **Секрет никогда не касается репо** — ни кода, ни конфигов, ни доков, ни брифов.
   `.env*` игнорится (managed-блок скелета), в репо живут только `*.example`-шаблоны
   с пустыми значениями.
2. **Секрет не касается терминала** — не `cat`-ать файлы с токенами, не эхать env.
   Вывод терминала попадает в логи сессий и контекст агентов (инцидент 2026-07-09:
   PAT засветился через npm-warning из-за кривой строки `~/.npmrc` → ревок).
3. **Каждому секрету — минимальный скоуп и короткая жизнь.** PAT — только
   `read:packages`; ротация при любом подозрении.

## Где что живёт

| Секрет | Место | Кто читает |
|---|---|---|
| PAT `read:packages` (GH Packages npm) | user-level `~/.npmrc`: `//npm.pkg.github.com/:_authToken=<PAT>` | локальный `pnpm install` (GH Packages требует auth даже на публичные npm-пакеты) |
| CI-доступ к пакетам | автоматический `GITHUB_TOKEN` (скоуп на один run) + grant «Manage Actions access» на пакете | reusable node-ci devopser'а |
| Runtime-конфиг backend | env-переменные `BRAINER_*` (endpoint OTEL, data-dir, буфер) — не секреты, но тот же канал | `packages/backend/app/config.py` |
| Auth Claude Code CLI | профиль CLI (`CLAUDE_CONFIG_DIR`); при мульти-акке §5 — по конфиг-директории на акк | адаптер claude-code |

В CI PAT'ы не живут: всё, что нужно workflow'ам, — `GITHUB_TOKEN` или GH Actions
secrets (пер-репо, добавлять осознанно, называть по потребителю).

## Защита от утечки (упреждающая)

- **GitHub secret scanning + push protection** — включить в Settings → Code security
  на каждом omnifield-репо (бесплатно для публичных). Ловит токен ДО попадания в историю.
- **Скан истории** при подозрении:
  `git grep -I -E "(ghp_|github_pat_|AKIA|_authToken=|BEGIN .*PRIVATE KEY)" $(git rev-list --all)`.
  brainer просканирован при переводе в public (2026-07-09) — чисто.
- gitleaks-степ в reusable CI — предложено devopser'у (их зона, см. consumer-brainer координацию).

## Инцидент-флоу (секрет засветился)

1. **Ревок немедленно** — до любых разбирательств (утёкший токен = мёртвый токен,
   даже если «вроде никто не видел»).
2. Выпуск нового с тем же минимальным скоупом; обновить место хранения.
3. Скан истории (команда выше). Если секрет в истории публичного репо — считать
   скомпрометированным навсегда; переписывание истории (`git filter-repo` + force-push)
   делает только architect и только как гигиену, не как «отмену» утечки.

## Девопсер-деплой (горизонт)

При переезде запуска под devopser секреты приезжают env-инжектом из его механизма
(GH Environments / vault) — файлы с секретами на диске управляемых репо не появляются.
Зафиксировать в брифах деплоя devopser'а.
