# Control Panel — как пользоваться

Пульт над agent-сессиями: запуск owner/architect-сессий headless (без терминала),
живой чат с сессией, статусы флота. Это догфуд-инструмент — им ведём owner'ов
всех продуктов, включая сам brainer.

## Запуск стека

Два процесса (по терминалу на каждый, или в фоне):

```bash
# 1. Backend (API + спавн сессий) — :8000
cd packages/backend
uv run uvicorn app.main:app --port 8000

# 2. Frontend (dev-сервер) — :5173, проксирует API на :8000
cd packages/frontend
pnpm dev
```

Открыть **http://localhost:5173**.

Prereq: Claude Code CLI установлен и авторизован (сессии спавнятся через него,
headless). Реестр сессий — sqlite в `%LOCALAPPDATA%/omnifield/brainer`, поэтому
сессии переживают рестарт backend'а (resume-on-start).

## Экраны

### Fleet (главный)

Список всех сессий: зона, репо, статус, время старта. Статусы:

| Статус | Значит |
|---|---|
| `Starting` | сессия поднимается (init) |
| `Running` | агент работает (пишет / думает / зовёт инструменты) |
| `Waiting` | ход закончен, агент ждёт твоего сообщения — **обычное состояние для «продолжай диалог»** |
| `Stopped` | сессия остановлена / стрим закрылся |

Клик по строке → чат сессии.

### Launch

Спавн новой сессии:

- **Repository** — управляемый репо (`omnifield/brainer`, `omnifield/writer`).
- **Scope** — зона = роль: `main` → architect (полный git), остальные → owner
  (commit-only, git-gate). Это уходит в `OMNIFIELD_SCOPE`, хуки репо отработают
  как при ручном `claude-scope.ps1`.
- **Model** (опц.) — пусто = дефолт; для дешёвых проверок — haiku
  (`claude-haiku-4-5-20251001`).
- **Brief path** (опц.) — путь к брифу; сессия сразу получит «прочитай бриф и
  приступай». Без брифа — сессия молча ждёт первого сообщения в чате.

«Spawn session» → сразу попадаешь в чат новой сессии.

### Чат сессии

- **Лента** — живой стрим: реплики агента, свёрнутые `thinking`-блоки (клик —
  раскрыть), пары tool-call/result (клик — вход/выход инструмента), статусы,
  `done`-строка с токенами и стоимостью хода.
- **Send-бокс** — сообщение в живую сессию, агент отвечает в том же контексте.
  Пока `Running` — идёт работа, дождись `done`/`Waiting`.
- **Stop** — мягкий interrupt текущего хода (сессия остаётся жива для follow-up).
- Обрыв соединения лечится сам: EventSource переподключается и доигрывает
  пропущенное без дублей.
- `error` / `limit` — видны в ленте (limit = упёрлись в квоту акка/рейт).

## Типовой флоу (догфуд)

1. Architect в своей сессии режет бриф → коммит в `briefs/`.
2. Launch: repo + scope зоны + Brief path → owner-сессия стартует headless.
3. Следишь за лентой (thinking/tool-вызовы), отвечаешь на вопросы owner'а
   прямо в чате.
4. `Waiting` + done-строка = ход завершён. Ревью — architect-сессией.
5. Stop, когда сессия больше не нужна.

## Известные ограничения (fixup'ы в работе)

- **Hard-refresh (F5) на странице сессии ломается** (прокси отдаёт 404 бэка) —
  заходи через Fleet, client-side. Фикс — Ф1 ревью.
- **Твои реплики после переподключения исчезают из ленты** (они пока не в wire) —
  контекст агента их НЕ теряет, только отображение. Фикс — Ф2 ревью.
- Статус в шапке чата может отставать от списка до следующего события (Ф3).
- История событий — в пределах жизни backend-процесса (ring-буфер): после
  рестарта backend лента начнётся с новых событий, контекст сессии жив.

## Fallback

Терминальный путь никуда не делся: `.\claude-scope.ps1 -Scope <зона>` в корне
управляемого репо — та же сессия руками, мимо панели.
